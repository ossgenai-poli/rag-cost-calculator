# Export Acceptance Specification

Authoritative spec for the three export buttons, for deterministic QA acceptance (test-plan
Suite L: L5–L7). Source of truth: `lib/share.ts` (`inputsToCsv`, `assumptionsToJson`, `buildReport`)
+ `app/page.tsx` download wiring. All exports are generated **client-side** from the current inputs
and the loaded price book; nothing is fetched at export time.

Common acceptance criteria (all three):
- The download is triggered by the toolbar button and a "… exported ✓" toast appears.
- The file is non-empty and parses/opens without error in a standard tool.
- Numbers match what's on screen for the **active** scenario at export time (API vs self-hosted).
- No `NaN`, `Infinity`, blank, or `$-` values.

---

## 1. CSV — `rag-cost-breakdown.csv`  (button: **Export CSV**)
MIME `text/csv`. Two blocks separated by one empty line. Values are RFC-4180 quoted only when they
contain `, " \n` (via `csvEscape`).

**Block A — headline metrics** (header row `Metric,Value`):
| Metric | Format |
|---|---|
| `Estimated monthly cost (USD)` | 2 decimals |
| `Cost per query (USD)` | 6 decimals |
| `Cost per 1,000 queries (USD)` | 2 decimals |
| `Annualized cost (USD)` | 2 decimals |
| `Monthly LLM tokens` | integer |
| `Monthly input tokens` | integer |
| `Monthly output tokens` | integer |
| `Queries per month` | integer |

**Block B — cost breakdown** (header row `Component,Monthly cost (USD),Share (%)`): one row per
breakdown component, in this order — `Ingestion (embedding)`, `Vector store (OpenSearch Serverless)`,
`Reranking`, the generation line (`Generation (LLM API)` **or** `GPU infrastructure (self-hosted LLM)`
depending on mode), `Guardrails`, `Query overhead (query embedding + infra)`, `Operations & overhead`.
Monthly = 2 decimals; Share = 2 decimals (percent, sums to ~100).

**Accept if:** both blocks present; headline `Estimated monthly cost` equals the on-screen monthly
total; breakdown Monthly values sum to the total (±0.01); Share sums to ~100%.

## 2. JSON — `rag-assumptions.json`  (button: **Export JSON**)
MIME `application/json`, pretty-printed (2-space). Top-level shape:
```jsonc
{
  "exportedFor": "AWS RAG Price Calculator",
  "pricing": {
    "asOf": "YYYY-MM-DD",
    "region": "us-east-1",
    "source": "live" | "fallback",
    "updatedAt": "<ISO timestamp>",
    "models": [ { "id", "label", "kind", "inPricePer1K", "outPricePer1K", "verifiedAt" }, … ],
    "opensearch": { "ocuPricePerHr", "storagePricePerGBmo", "gbRamPerOcu", "minOCU" },
    "gpus": [ { "instanceType", "gpu", "pricePerHr", "sustainedTokPerSec", "totalMemGB" }, … ]
  },
  "inputs": { /* the FULL CalcInputs object: corpus, chunking, vectorStore, retrieval,
                 guardrails, generation, managedKb, ops, traffic, queryTokens, ragMode */ }
}
```
**Accept if:** valid JSON; `pricing.models` includes the current catalog (the 5 open-weight models +
Bedrock models — **not** obsolete ids like `qwen2.5-72b-bedrock`); `pricing.gpus` = `p5.48xlarge`,
`p5e.48xlarge`, `p6-b200.48xlarge` (no `p4d`/`g5`); `inputs` round-trips (loading it reproduces the
same on-screen total). `source` reflects reality (`fallback` on the static export, `live` only when
`/api/prices` returned live AWS data).

## 3. Markdown report — `rag-cost-report.md`  (button: **Export report**)
MIME `text/markdown`. Sections, in order:
1. `# RAG Cost Report` + bullet metadata: **Region**, **Pricing as of** (`… (source: …)`),
   **Deployment** (`Self-hosted GPU · <model>` or `Bedrock API · <model>`), **Traffic**
   (`<n> queries/mo` and, if peak > 1, `<x>× peak-to-average`).
2. `## Headline` — table: Estimated monthly cost, Cost per query, Cost per 1,000 queries, Annualized,
   Largest driver (`<label> (<share>%)`).
3. `## Cost breakdown (monthly)` — table `Component | Monthly | Share`, ending with a bold **Total** row.
4. `## Scenario comparison` — table `Scenario | Monthly | Per 1K | vs baseline` for the 4 scenarios
   (Self-built + API, Bedrock KB + API, Self-built + GPU, GPU at break-even). `—` where a value is N/A.
5. `## Self-host vs API crossover` — Verdict; and when break-even exists, Break-even tokens/mo (~QPS)
   and Utilization to break even (`N%` or `N× capacity (infeasible)`).
6. `## Key assumptions` — self-hosted: GPU instance + on-demand $/hr + purchasing model + uptime;
   fleet (instances / memory floor / throughput need); precision + ctx × concurrency. API: model +
   in/out $/1K. Plus retrieval (topK/topN/rerank), guardrails (on/off), and an ops line when non-zero.
7. Trailing italic: `_Generated by the RAG Cost Calculator. Figures are planning estimates …_`

**Accept if:** all seven sections present; Total in §3 equals the on-screen monthly total; the
scenario table matches the on-screen comparison; self-hosted reports name the purchasing model + uptime.

## Download harness note (for the tester)
The in-app browser used during initial testing may not surface `blob:` downloads. Validate downloads
with a real browser (Chrome/Firefox/Safari/Edge) or drive them via Playwright (`page.on('download')`).
The buttons call `downloadText(filename, content, mime)` which creates a `Blob` + object URL and clicks
a synthetic `<a download>` — standard and Playwright-observable.
